# 新增网站操作手册

这份手册用于指导你把一个新的招标网站接入到当前监控系统中。

适用场景：

- 政府采购网
- 公共资源交易中心
- 行业招标公告站
- 以 HTML 列表页展示公告的网站

不适用场景：

- 页面内容完全靠前端接口动态加载，初始 HTML 里没有公告数据
- 需要登录、验证码、签名参数或复杂 Cookie 才能看到列表
- 网站做了严格反爬，普通浏览器能看，但程序请求拿不到内容

---

## 一、先理解 3 个选择器

系统新增网站时，最常用的是下面 3 个字段：

### 1. `list_selector`

表示“公告列表项”选择器。

它应该命中：

- 一条公告对应一个节点
- 节点里至少包含标题链接
- 最好节点里还能包含日期

常见示例：

```css
ul.notice-list li
table.list tr
div.article-list li
div.mainList.tzggList li
```

不要优先写成非常宽泛的：

```css
a
div a
```

这种写法容易把导航、菜单、分页、登录入口一起抓进来。

### 2. `title_selector`

表示“相对于列表项，标题在哪”。

如果 `list_selector` 已经直接命中 `<a>` 标签，`title_selector` 通常可以留空，或者填：

```css
a
```

如果 `list_selector` 命中的是 `li`、`tr`、`div`，标题在里面某个位置，就填：

```css
a
.title a
td a
```

### 3. `date_selector`

表示“相对于列表项，发布日期在哪”。

常见示例：

```css
span
.date
.publish-time
td:last-child
```

系统会自动识别类似下面的日期格式：

- `2026-04-21`
- `2026/04/21`
- `2026年4月21日`
- `2026-04-21 09:30`

如果列表页本身没有日期，可以先留空，但展示效果会比有日期差。

---

## 二、新增网站的标准流程

建议每次都按下面的顺序操作：

1. 打开目标网站的“公告列表页”，不是首页。
2. 在浏览器里确认这一页能直接看到公告标题。
3. 打开系统的 **网站管理**。
4. 点击 **添加网站**。
5. 先填网站名称和列表页 URL。
6. 再填写 `list_selector`。
7. 如果标题或日期不稳定，再补 `title_selector` 和 `date_selector`。
8. 点击 **校验选择器**。
9. 看到命中结果合理后，再点击 **保存**。
10. 保存后手动点一次 **抓取**，去 **招标公告** 页面确认实际结果。

记住一个原则：

- 先让它“能抓到”
- 再让它“抓得准”

不要一开始就追求最复杂的精细选择器。

## 三、怎么找到正确的选择器

下面用 Chrome、Edge 这类浏览器举例。

### 第一步：找到公告标题

1. 打开目标网站列表页
2. 在某一条公告标题上点鼠标右键
3. 选择 **检查**
4. 浏览器会定位到对应的 HTML 节点

常见情况有两种。

#### 情况 A：直接定位到 `<a>`

例如：

```html
<a href="/notice/123.html">某某项目招标公告</a>
```

说明标题节点本身就是链接，这种最简单。

你需要继续往上看它属于哪个列表项：

```html
<li>
  <a href="/notice/123.html">某某项目招标公告</a>
  <span>2026-04-21</span>
</li>
```

这时通常可以写：

```text
list_selector: ul.notice-list li
title_selector: a
date_selector: span
```

#### 情况 B：标题在更深层结构里

例如：

```html
<li>
  <div class="title">
    <a href="/notice/123.html">某某项目招标公告</a>
  </div>
  <div class="date">2026-04-21</div>
</li>
```

这时可以写：

```text
list_selector: ul.notice-list li
title_selector: .title a
date_selector: .date
```

### 第二步：确认“一条公告对应一个列表项”

这是最重要的一步。

你要找的是“每条公告重复出现的最小外层结构”，通常是：

- `li`
- `tr`
- 某个重复的 `div`

如果你发现页面结构像这样：

```html
<ul class="notice-list">
  <li>...</li>
  <li>...</li>
  <li>...</li>
</ul>
```

那么优先用：

```css
ul.notice-list li
```

如果结构像这样：

```html
<table class="list">
  <tr>...</tr>
  <tr>...</tr>
  <tr>...</tr>
</table>
```

那么优先用：

```css
table.list tr
```

不要直接把整个大容器当成 `list_selector`，否则系统可能把多条公告拼成一条。

### 第三步：找日期节点

在同一个列表项里找时间。

常见写法：

```html
<span>2026-04-21</span>
<td>2026-04-21</td>
<div class="date">2026-04-21</div>
```

如果日期就在列表项里唯一的 `<span>`，可以直接填：

```css
span
```

如果有多个 `<span>`，就要更具体一点：

```css
.date
span.time
td:last-child
```

## 四、怎么判断当前配置“对不对”

点击 **校验选择器** 后，重点看下面 4 件事。

### 1. 命中数量是否合理

如果页面肉眼能看到 10 条公告，而系统只命中 1 条或 0 条，说明选择器太窄或写错了。

如果页面只有十几条公告，系统却命中上百条，说明选择器太宽，把菜单、栏目页、分页、历史链接一起抓进来了。

### 2. 样例标题是否真的是公告

如果校验结果里出现这些内容，通常说明选择器不够准：

- 首页
- 更多
- 下一页
- 通知公告
- 登录入口
- 统一数字招标系统

### 3. 日期是否能识别

如果样例里标题是对的，但发布日期一直显示“未识别”，说明：

- `date_selector` 没写对
- 或者列表页根本没有日期

### 4. 链接点开后是否进入正文

如果点开样例链接进入的是：

- 栏目页
- 首页
- 系统登录页
- 搜索页

说明当前抓到的不是正文链接，要重新收窄 `list_selector` 或 `title_selector`。

## 五、常见配置模板

### 模板 1：普通公告列表

HTML 结构：

```html
<ul class="notice-list">
  <li>
    <a href="/detail/1.html">公告标题</a>
    <span>2026-04-21</span>
  </li>
</ul>
```

建议配置：

```text
list_selector: ul.notice-list li
title_selector: a
date_selector: span
```

### 模板 2：标题带 class

HTML 结构：

```html
<li>
  <div class="title"><a href="/detail/1.html">公告标题</a></div>
  <div class="date">2026-04-21</div>
</li>
```

建议配置：

```text
list_selector: li
title_selector: .title a
date_selector: .date
```

### 模板 3：表格列表

HTML 结构：

```html
<table class="list">
  <tr>
    <td><a href="/detail/1.html">公告标题</a></td>
    <td>2026-04-21</td>
  </tr>
</table>
```

建议配置：

```text
list_selector: table.list tr
title_selector: td a
date_selector: td:last-child
```

### 模板 4：只想抓特定类型公告

如果一个列表里同时有“通知、招标公告、成交公告、需求公示”，你只想抓其中一类，就要继续收窄。

例如：

```css
div.mainList.tzggList li a[href*="articleId="]
```

这种写法适合在你已经确认目标链接有明显特征时使用。

## 六、常见问题怎么排查

### 1. 校验结果是 0 条

按下面顺序检查：

1. URL 是否真的是公告列表页
2. 页面是否能直接在 HTML 里看到公告，不是纯前端渲染
3. `list_selector` 是否写错
4. 页面是否需要登录、验证码、Cookie

### 2. 校验结果很多，但大部分不是公告

说明 `list_selector` 太宽。

处理方法：

- 从 `a` 往上找真正重复的 `li/tr/div`
- 把选择器写得更具体
- 必要时增加属性限制

例如：

```css
div.mainList.tzggList li a[href*="articleId="]
```

### 3. 标题正确，但日期为空

说明 `date_selector` 没命中。

处理方法：

1. 在同一个列表项中重新定位日期节点
2. 尽量不要写全局选择器
3. 优先写相对于列表项的短路径

### 4. 能抓到，但链接点开不对

说明抓到的是栏目链接或导航链接，不是正文链接。

处理方法：

- 把 `list_selector` 收窄到真正的公告列表
- 或者把 `title_selector` 指向正文标题链接

### 5. 浏览器能看，系统抓不到

常见原因：

- 网站依赖前端接口动态渲染
- 网站要求登录或验证码
- 目标站点启用了反爬
- 需要代理、Cookie 或额外请求头

这种情况通常不是改选择器能解决的，需要程序层适配。

---

## 七、什么时候只配选择器就够了

下面这些情况，一般不需要改代码：

- 页面源码里直接有公告列表
- 标题是普通 `<a>`
- 日期就在同一条列表项里
- 列表页不需要登录
- 手动打开页面和程序请求看到的内容一致

---

## 八、什么时候应该找人改代码

出现下面任一情况时，优先判断为“需要代码适配”：

- 页面初始 HTML 基本为空，数据是 JS 后加载的
- 站点接口返回 JSON，不是普通 HTML 列表
- 需要验证码、登录态、Token、签名参数
- 需要固定 Cookie 才能看到内容
- HTTPS/SSL 异常
- 浏览器能看，程序请求被重定向到别的页面
- 日期格式非常特殊，系统无法识别

如果你遇到这些情况，最好把下面信息一起记录下来：

- 目标网站 URL
- 你尝试过的 3 个选择器
- 校验结果截图
- 浏览器中该公告的 HTML 片段
- 是否需要登录

这样后续排查会快很多。

---

## 九、推荐的实际操作习惯

每新增一个网站，建议都按这个习惯来：

1. 先保存你尝试过的选择器
2. 每次只改一个字段
3. 先看“校验选择器”
4. 再看“实际抓取”
5. 最后看“公告列表”里的最终效果

不要同时改 URL、列表选择器、标题选择器、日期选择器四样东西，否则很难判断到底是哪一步出了问题。

---

## 十、新增网站前的检查清单

在保存前，至少确认下面这些点：

- 列表页 URL 正确
- `list_selector` 命中的是一条条公告
- 样例标题不是菜单或导航
- 链接点开能进入正文
- 日期能识别，或者你确认该列表页没有日期
- 校验结果数量与页面可见数量大致一致

如果这 6 项都通过，通常就可以保存并开始抓取。

---

## 十一、一句话经验

新增网站最重要的不是把选择器写得“复杂”，而是把它写得“刚好命中每一条公告”。

优先选“每条公告的重复外层结构”，再从里面拆标题和日期，成功率最高。
