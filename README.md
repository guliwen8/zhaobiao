# 企业投标管理系统

面向本地单机部署的招标公告监控与投标过程管理工具，适合投标团队或内部业务团队从“发现公告”推进到“项目跟进、任务分工、资料复用、标书草稿输出”。

系统特点：

- `FastAPI + APScheduler + SQLite` 轻量部署，无需额外数据库
- 支持自定义列表、标题、日期 CSS 选择器
- 支持手动抓取、定时抓取、关键词过滤、显式已读
- 新增投标项目库、任务看板、企业资料库、标书内容模板
- 支持从公告一键转入投标项目，并自动生成资格审查等默认任务
- 支持按企业资料和模板导出 Markdown 标书方案初稿
- 自带健康检查、选择器预览、抓取统计和失败告警，便于演示与交付验收


## 启动方式

### 1. 启动后端

```bash
python3 main.py
```

启动后默认监听：

- API 根地址：`http://localhost:8000/api`
- 健康检查：`http://localhost:8000/api/health`

### 2. 打开前端

推荐访问后端直接托管的页面：

```text
http://localhost:8000
```

也可以直接用浏览器打开当前目录下的 `index.html`。通过后端访问时，前端会自动使用同源 API；通过本地文件打开时，默认请求 `http://localhost:8000/api`。后端已默认允许：

- `file://` 打开的页面（浏览器表现为 `Origin: null`）
- `http://localhost:*`
- `http://127.0.0.1:*`

## 核心功能

### 网站管理

- 添加、编辑、启停、删除监控网站
- 配置列表页、标题、日期选择器
- 保存前可执行“校验选择器”，查看命中数量和样例标题

### 招标公告

- 按网站、关键词、时间范围筛选
- 支持分页查看和手动删除
- `NEW` 标签仅在显式“标记已读”后清除，不会因为打开列表自动消失
- 支持将公告一键转入投标项目库

### 投标项目

- 管理项目名称、客户、编号、阶段、负责人、截止日期、预算金额和中标概率
- 统计项目总量、推进中项目、7 天内截止项目、逾期任务和启用模板数
- 支持阶段筛选、关键词搜索、项目编辑、删除和标书草稿导出

### 任务看板

- 按待办、进行中、已完成、已取消展示投标任务
- 支持负责人、截止日期、优先级、说明维护
- 支持快速标记完成，逾期任务自动高亮

### 企业资料与模板

- 维护企业名称、联系人、资质能力、核心优势、服务承诺、相关案例
- 管理商务、技术、服务等标书内容模板
- 导出草稿时会自动组合项目资料、企业资料、模板内容和任务计划

### 关键词过滤

- 支持单条或批量录入
- 支持使用 `,`、`，`、`、` 分隔多个关键词
- 数据库存储为“一行一个关键词”

### 调度与健康检查

- 查看每个站点的下次执行时间
- 查看调度器运行状态、启用站点数、公告总数
- 支持立即触发全量抓取
- 支持查看各站点抓取成功率、失败次数、平均耗时和最近 7 天失败告警

## 对外交付建议流程

建议演示顺序：

1. 打开 `index.html`
2. 进入“网站管理”，新增站点并点击“校验选择器”
3. 保存站点后执行“抓取”
4. 进入“招标公告”验证发布日期、来源网站和 `NEW` 标签
5. 点击公告行内“转项目”，进入“投标项目”查看默认任务
6. 进入“任务看板”维护负责人、截止日期和状态
7. 进入“企业资料”补充企业资质、优势、服务承诺和模板
8. 在“投标项目”点击“草稿”导出 Markdown 标书方案初稿
9. 进入“调度状态”展示任务计划和健康检查

## 关键接口

- `GET /api/health`：服务健康检查
- `GET /api/stats`：首页统计
- `GET /api/sites`：站点列表
- `POST /api/sites/preview`：选择器校验预览
- `POST /api/sites/{id}/crawl`：手动抓取单站点
- `POST /api/crawl-all`：触发全量抓取
- `GET /api/bids`：公告查询
- `POST /api/bids/mark-read`：按筛选条件批量标记已读
- `GET /api/keywords`：关键词列表
- `GET /api/enterprise/dashboard`：投标项目经营看板
- `GET /api/projects`：投标项目列表
- `POST /api/projects/from-bid/{bid_id}`：公告转投标项目
- `GET /api/projects/{id}/proposal`：导出标书方案初稿
- `GET /api/tasks`：任务列表
- `GET /api/company-profile`：企业资料
- `GET /api/templates`：标书模板库

## 数据文件说明

- 默认数据库文件：`bid_monitor.db`
- 数据库存储在当前运行目录，可直接备份或替换
- 首次启动会自动创建表，并补充内置示例站点
- 升级后会自动增量创建 `bid_projects`、`project_tasks`、`bid_templates`、`company_profile` 等企业投标管理表

## 可配置项

通过环境变量可调整运行行为：

- `BID_MONITOR_DB_PATH`：SQLite 路径，默认 `bid_monitor.db`
- `BID_MONITOR_ALLOWED_ORIGINS`：允许的精确来源，逗号分隔
- `BID_MONITOR_ALLOWED_ORIGIN_REGEX`：允许的来源正则，默认仅限本机地址
- `BID_MONITOR_USE_ENV_PROXY`：是否使用系统代理环境变量，默认关闭；仅在必须走代理时设为 `1`
- `ANTHROPIC_API_KEY`：仅根目录旧版 AI 标书生成脚本需要，企业部署时请通过环境变量配置，不要写入代码文件

## 常见问题

### 1. 页面提示无法连接后端

确认 `main.py` 已启动，且端口仍为 `8000`。

### 2. 选择器命中 0 条

先在浏览器开发者工具中确认：

- 列表选择器是否命中公告容器
- 标题选择器是否相对于列表项正确
- 日期选择器是否真的包含发布日期

### 3. 发布日期为空

说明目标页面里未匹配到日期元素，或页面本身没有在列表页展示发布时间。优先补全 `date_selector`。

### 4. 部分站点抓取失败

部分政务网站可能有反爬、验证码、登录态或地域访问限制。此时需要：

- 调整请求头
- 增加 Cookie 或鉴权
- 改为抓取可公开访问的公告列表页

## 注意事项

- 本系统定位为单机使用，不包含登录、多租户、消息队列等能力
- 数据抓取结果依赖目标网站结构，网站改版后需重新调整选择器
- 请合理设置抓取频率，避免对目标网站造成不必要压力
